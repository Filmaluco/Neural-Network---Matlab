O programa vai dar como output uma "class" representada por um vector:
[0 1 0] -> neste caso sera da class 2 (Iris Virginica)

lembrar do exemplo de exame, por cada possibilidade de saida temos um "neurao" apenas um vai ficar ligado


overfiting e' basicamente perder a capacidade de generalizacao, fica tao habituada aos dados (que contem ruido)
que depois nao consegue se adpatar a outros dados.... para evitar isto divide-se os dados em:

Treino
Validacao (os que sao usados para validacao nao sao os mesmos usados para treino, servem para corrigir o overfiting)
Teste

Funcoes de treino:
 - traingd
 - traingdx
 - trainbfg
 - trainrp (bom para imagens)

Funcoes de activacao:
 - tansing
 - logsig
 - purelin
 - tansig (bom para classificacoes)

 camadas intermedias em feedfoward: tansig

 camadas escondidas nao convem ter funcoes lineares! Deve ser usado para output mas nem ai e' sempre melhor
